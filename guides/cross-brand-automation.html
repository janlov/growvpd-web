Cross-Brand Automation: Mixing Tuya, Mars Hydro & More

The Problem: Walled Gardens

Every smart device brand has its own app, its own cloud, and its own automation system. A Tuya sensor can trigger a Tuya plug within the Tuya app. An AC Infinity controller can adjust an AC Infinity fan. But what if you want a Tuya sensor to trigger a Mars Hydro fan? Or an AC Infinity temperature reading to turn on a Tuya smart plug connected to your humidifier?

None of these brands talk to each other natively. Their apps are walled gardens. This forces growers to either commit to a single ecosystem (limiting their equipment choices) or run multiple apps with manual coordination (which defeats the purpose of automation).

The Solution: GrowVPD Pro as the Hub

GrowVPD Pro connects to multiple platforms simultaneously and treats all devices as part of one unified system. It reads sensor data from any connected platform and can trigger actions on any connected device, regardless of brand. The app acts as the central intelligence layer that sits above all the individual ecosystems.

Supported platforms:

  • Tuya / Smart Life — sensors, smart plugs, switches, cameras, dimmers, AC units, irrigation
  • AC Infinity — inline fans, clip fans, controllers with integrated sensors
  • Mars Hydro — LED drivers, inline fans, clip fans (WiFi + MQTT)
  • Spider Farmer — LED drivers, fans, sensors (WiFi + MQTT, shared protocol with Mars Hydro)
  • TP-Link Tapo — cameras (ONVIF snapshots + RTSP)

Real-World Cross-Brand Scenarios

Scenario 1: Tuya Sensor + Mars Hydro Exhaust Fan

You have a cheap Tuya WiFi temperature/humidity sensor ($15) in your tent and a Mars Hydro 6-inch inline fan. You want the fan to speed up when VPD exceeds your target.

1

Setup

  • Connect your Tuya account in Settings (Access ID + Secret)
  • Connect your Mars Hydro account in Settings (email + password)
  • Both the Tuya sensor and Mars Hydro fan appear in the Automation tab
  • Assign both to the same tent
2

Create the Rule

  • Sensor: Tuya WiFi Sensor (reads temperature + humidity)
  • Condition: VPD > 1.4 kPa
  • Action (trigger): Set Mars Hydro fan to speed 8
  • Action (restore): Set Mars Hydro fan to speed 4
  • Hysteresis: 0.15 kPa

Result: The Tuya sensor feeds VPD data to GrowVPD Pro. When VPD crosses 1.4 kPa, the app sends an MQTT command to the Mars Hydro fan to increase speed. When VPD drops below 1.25 kPa (1.4 - 0.15 hysteresis), it reduces the fan back to speed 4. Two different brands, one seamless rule.

Scenario 2: AC Infinity Controller + Tuya Humidifier Plug

Your AC Infinity Controller 69 Pro has a built-in temperature/humidity sensor. Your humidifier is plugged into a Tuya smart plug.

1

The Rule

  • Sensor: AC Infinity Controller 69 Pro (temp + RH)
  • Condition: VPD > 1.2 kPa (air too dry for veg)
  • Action (trigger): Turn ON Tuya smart plug (humidifier)
  • Action (restore): Turn OFF Tuya smart plug
  • Min on-time: 5 minutes (prevents rapid cycling)

Scenario 3: Multi-Sensor VPD Averaging

Large tents can have significant climate variation between the canopy center and the edges. Place two sensors from different brands (e.g., one Tuya, one AC Infinity) and GrowVPD Pro uses the tent's primary sensor for automation while displaying all readings for comparison.

Scenario 4: Camera Timelapse from Mixed Brands

You have a Tuya camera in Tent A and a Tapo camera in Tent B. Both capture daily snapshots that auto-save to each tent's grow diary. The timelapse generator works with photos from either camera brand.

Key insight: GrowVPD Pro does not care which brand provides the sensor data or which brand receives the command. A VPD rule is a VPD rule — the sensor source and the actuator target are independently configurable.

Setting Up Cross-Brand Automation

  1. Connect all your platform accounts in Settings > Connected Accounts. You can have Tuya, AC Infinity, Mars Hydro, and Tapo all connected simultaneously.
  2. Discover devices from each platform in the Automation tab. All devices from all platforms appear in a single discovery list.
  3. Organize into rooms and tents. A single tent can contain devices from multiple platforms.
  4. Create automation rules. When creating a rule, the sensor dropdown lists all sensors from all platforms, and the target device dropdown lists all controllable devices from all platforms.
  5. The automation engine handles the rest. It polls sensors every 15 minutes (via AutomationWorker), evaluates rules, and sends commands to the correct platform API for each target device.
Note: Cross-brand automation requires a Pro subscription. All platform connections and automation rules are Pro features. The free version shows device data in demo mode with sample data.

Troubleshooting Cross-Brand Rules

  • Rule not triggering: Check that both the sensor device and the target device show as "online" in the Automation tab. If either is offline, the rule cannot execute.
  • Delayed response: GrowVPD Pro polls sensors every 15 minutes. A VPD spike that occurs and resolves within a single polling interval may not trigger a rule. For faster response, ensure your sensor platform reports frequently.
  • Token expired: Each platform has its own authentication token. If one expires, devices from that platform go offline. Re-authenticate in Settings > Connected Accounts.
  • Conflicting rules: If two rules target the same device with opposing actions, the higher-priority rule wins. Check rule priorities in the Automation tab.
